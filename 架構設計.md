# 體重管理師網站 — 架構設計文件

> **定位**：行銷型招募網站，目標是讓潛在學員了解服務內容、建立信任感，並主動聯繫報名。  
> **技術**：純 HTML / CSS / JavaScript（靜態網站，無需後端）

---

## 一、網站目標與目標族群

### 核心目標
- 吸引對體重、體脂、代謝有困擾的潛在學員
- 清楚傳達四大服務的差異化價值
- 降低諮詢門檻，讓訪客主動填表或加 LINE

### 目標族群
| 族群 | 痛點 |
|------|------|
| 25–45 歲上班族 | 久坐、代謝下降、體脂偏高 |
| 體檢紅字困擾者 | 血脂、血糖、尿酸異常但不知如何改善 |
| 曾節食失敗者 | 體重反彈、肌肉量流失 |
| 健身停滯者 | 長期運動但體脂未降 |

---

## 二、頁面架構（Sitemap）

```
首頁 (index.html)
├── 我的故事 / 關於我 (about.html)
├── 服務內容 (services.html)
│   ├── 體重體脂管理
│   ├── 飲食份量建議
│   ├── 體檢報告解讀
│   └── 運動訓練規劃
├── 成果案例 (results.html)
├── 常見問題 (faq.html)
└── 聯絡 / 報名 (contact.html)
```

## 三、各頁面內容規劃

### 1. 首頁 `index.html`

**目的**：在 5 秒內讓訪客知道「這裡能幫我什麼」並引發興趣。

#### 區塊設計（由上到下）

| 區塊 | 內容 | 關鍵元素 |
|------|------|---------|
| **Hero（主視覺）** | 大標語 + 副標 + CTA 按鈕 | 背景圖片（健康、數據感）、主色調 |
| **數字亮點** | 幾位學員、平均減脂 X kg、最長追蹤 X 週 | 3 個數字橫排 |
| **問題共鳴** | 「你是否有這樣的困擾？」列表 | 勾選清單樣式 |
| **四大服務速覽** | 4 張服務卡片，各有圖示 + 一句話說明 + 「了解更多」 | 連結至 services.html |
| **成果預覽** | 1–2 位學員的 Before/After 數字對比 | 連結至 results.html |
| **學員見證** | 2–3 則文字引言 | 頭像（可匿名）、姓名縮寫 |
| **流程說明** | 報名 → 初步評估 → 制定計劃 → 定期追蹤 | 步驟條 |
| **最終 CTA** | 「立即預約免費諮詢」 | 跳至 contact.html 或彈窗 |

**主標語建議**：
```
「不只是少吃，是讓身體重新找回代謝平衡」
「科學數據 × 個人化計劃 × 長期陪伴」
```

---

### 2. 關於我 `about.html`

**目的**：建立信任感，讓學員覺得「這個人懂我、有能力幫我」。

#### 區塊設計

- **個人照片 + 一段話**：為什麼走上體重管理這條路（故事性）
- **專業背景 / 資格**：相關證照、學習歷程
- **服務理念**：「我相信的體重管理方式」，與一般節食減重做區別
- **使用工具 / 方法**：體脂秤（六合一測試儀）、飲食記錄、體檢數據追蹤
- **一句話信念**：作為頁面結語

---

### 3. 服務內容 `services.html`

**目的**：讓訪客清楚理解每項服務的內容、適合誰、能帶來什麼。

#### 四大服務詳細說明

##### 服務 1：體重體脂管理
- **說明**：每週定期量測體重、體脂率、肌肉率、內臟脂肪等 16 項指標，以圖表追蹤趨勢
- **適合**：想要科學化管理身材、避免肌肉流失的人
- **你會得到**：每週數據報告、趨勢圖、自動計算 BMI / 肌肉量 / 體脂重

##### 服務 2：飲食份量建議
- **說明**：根據目標體組成，每 3–4 天規劃早午晚餐的份量（P 蛋白質 / C 醣類 / F 脂肪）
- **適合**：不想計算卡路里、需要有人告訴我「吃什麼份量」的人
- **你會得到**：個人化飲食份量表，隨目標動態調整

##### 服務 3：體檢報告解讀
- **說明**：協助解讀血液檢查（血脂、血糖、肝腎功能、HbA1c、HOMA-IR 等），對應飲食與生活型態調整建議
- **適合**：體檢紅字多、不知道如何改善的人
- **你會得到**：各項數值趨勢追蹤，以及對應的飲食 / 生活建議

##### 服務 4：運動訓練規劃
- **說明**：依據體組成目標（增肌 / 減脂 / 維持），搭配訓練類型與頻率建議
- **適合**：長期運動但效果不明顯、或不知道怎麼搭配飲食的人
- **你會得到**：訓練方向建議，與飲食計劃整合

#### 收費方案區塊（選填）

| 方案 | 內容 | 建議對象 |
|------|------|---------|
| 基礎版 | 體重體脂追蹤 + 飲食份量 | 初次嘗試者 |
| 進階版 | 基礎版 + 體檢報告解讀 | 有健康數值困擾 |
| 全方位版 | 全部四項服務 | 認真長期改善者 |

### 4. 成果案例 `results.html`

**目的**：用真實數字建立「這真的有效」的信心。

#### 呈現方式

每個案例卡片包含：
- 學員代號（如「學員 A」）或姓名縮寫
- 基本資料：年齡、性別、追蹤週數
- **關鍵數字對比表格**：

| 指標 | 開始 | 目前 | 變化 |
|------|------|------|------|
| 體重 | 85.2 kg | 78.1 kg | −7.1 kg |
| 體脂率 | 32.4% | 26.8% | −5.6% |
| 肌肉率 | 55.2% | 60.1% | +4.9% |
| 內臟脂肪 | 14 | 10 | −4 |
| BMI | 29.1 | 26.7 | −2.4 |

- 學員一句話心得（引言格式）
- 追蹤週數折線圖（可選，用 Chart.js 繪製）

---

### 5. 常見問題 `faq.html`

**目的**：消除疑慮，減少諮詢前的障礙。

#### 問題分類

**關於服務**
- Q：一對一還是團體進行？
- Q：需要配合特定飲食法嗎（生酮、間歇？）
- Q：多久能看到效果？
- Q：如果出差或外食怎麼辦？

**關於費用**
- Q：收費方式？月繳 / 週繳？
- Q：可以先試用嗎？

**關於方法**
- Q：量體重需要用特定的秤嗎？
- Q：需要自己記錄食物嗎？
- Q：體檢報告需要上傳圖片嗎？

**關於效果**
- Q：我只想減脂但不想少肌肉，做得到嗎？
- Q：體重沒降但體脂有降，算成功嗎？

---

### 6. 聯絡 / 報名 `contact.html`

**目的**：讓有意願的訪客輕鬆踏出第一步。

#### 區塊設計

- **免費初步諮詢說明**：說明諮詢內容（幾分鐘、談什麼）
- **聯絡表單**（簡單）：
  - 姓名
  - 聯絡方式（LINE ID 或手機）
  - 主要困擾（下拉選單：體重過重 / 體脂過高 / 體檢紅字 / 運動停滯 / 其他）
  - 希望開始時間（本週 / 下個月 / 尚未確定）
  - 備註
- **直接聯絡方式**：
  - LINE 加好友按鈕（綠色）
  - 或 Email
- **服務地點 / 方式**：線上 / 實體（說明區域）

## 四、設計風格規範

### 色彩系統

| 用途 | 色碼 | 說明 |
|------|------|------|
| 主色（Primary）| `#2C7BE5` | 科技感、信任藍 |
| 強調色（Accent）| `#27AE60` | 健康綠，用於正向數字、CTA 按鈕 |
| 警示色 | `#E74C3C` | 僅用於需要改善的數值 |
| 背景 | `#F8FAFC` | 淺灰白，降低視覺疲勞 |
| 文字主色 | `#1A1A2E` | 深色，高對比 |
| 文字副色 | `#6C757D` | 說明文字 |

### 字型建議

```css
/* 主要字型 */
font-family: 'Noto Sans TC', sans-serif;  /* 繁體中文 */

/* 數字 / 數據 */
font-family: 'Inter', sans-serif;  /* 英文數字清晰 */
```

### 設計原則

- **數據感**：用卡片、標籤、百分比進度條呈現數字
- **信任感**：真實照片（可用圖示代替）、具體數字、學員回饋
- **行動引導**：每個頁面至少一個 CTA 按鈕，配色用強調綠
- **行動裝置優先**：主要瀏覽環境是手機（學員分享連結給朋友）

## 八、製作順序建議

1. **第一步** — 建立共用 CSS（色彩變數、字型、按鈕、卡片）
2. **第二步** — 製作導航列 + 頁尾（所有頁面共用）
3. **第三步** — 完成首頁 `index.html`（最重要）
4. **第四步** — 服務頁 `services.html`
5. **第五步** — 案例頁 `results.html`
6. **第六步** — 關於我 `about.html`
7. **第七步** — 常見問題 `faq.html`
8. **第八步** — 聯絡頁 `contact.html` + 表單驗證
